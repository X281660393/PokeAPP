# 宝可梦小图鉴 — 打包与部署说明

> 一个基于 Vue 3 + Capacitor 的跨平台宝可梦查询应用，可同时打包为**静态网站**与**安卓应用（APK）**。
> 已内置离线图片缓存（1025 张精灵图随包发布）与强制竖屏。

---

## 一、技术栈与使用的库

| 类别 | 库 / 工具 | 版本 | 用途 |
|------|-----------|------|------|
| 框架 | **Vue** | ^3.5.13 | 前端 UI 框架（Composition API + `<script setup>`） |
| 构建 | **Vite** | ^6.2.0 | 开发服务器 + 生产构建（极速打包） |
| 语言 | **TypeScript** | ~5.7.2 | 类型安全 |

| 状态 | **Pinia** | ^3.0.2 | 全局状态管理（搜索 / 筛选 / 详情） |
| UI | **Vant** | ^4.9.15 | 移动端组件库（已用内联 SVG 替代需联网的字体图标，保证离线可用） |
| 原生 | **@capacitor/core** | ^7.2.0 | 跨平台原生桥接核心 |
| 原生 | **@capacitor/android** | ^7.2.0 | 安卓平台适配（WebView 容器） |
| 原生 | **@capacitor/cli** | ^7.2.0 | Capacitor 命令行（`sync` / `open`） |
| 类型 | **vue-tsc** | ^2.2.0 | Vue + TS 类型检查（`npm run build` 前置） |

> **离线相关设计**
> - 所有图标改用**内联 SVG**（`SearchBox` 组件等），不依赖在线字体图标 CDN。
> - 1025 张宝可梦精灵图预下载至 `public/sprites/`，`spriteFor()` 返回相对路径 `./sprites/{id}.png`，构建后随 `dist/` 与 APK 一起发布，离线可用。
> - `vite.config.ts` 中 `base: './'` 保证网页在子路径 / `file://` 下也能正确加载资源。
> - 网站 favicon 为自绘精灵球 SVG（`public/pokeball.svg`），浏览器标签页显示精灵球图标。

## 三、打包为静态网站

网页是纯静态文件，构建产物 `dist/` 可部署到任意静态托管。

```bash
# 1. 安装依赖（首次或依赖变更时）
npm install

# 2. 类型检查 + 生产构建
npm run build

# 3. 本地预览（可选）
npm run preview
```

**产物**：`dist/` 目录（已含 1025 张本地精灵图）。

**部署方式（任选其一）**
- **CloudStudio 一键部署**：直接上传 `dist/` 并生成可访问 URL。
- **Vercel / Netlify**：拖拽 `dist/` 文件夹或关联 Git 仓库。
- **GitHub Pages**：将 `dist/` 推送到 `gh-pages` 分支。
- **自有服务器**：将 `dist/` 内容放到 Nginx / Apache 根目录。

> 由于图片已是相对路径且本地化，部署后**无需联网**即可显示宝可梦精灵图。

## 四、打包为安卓应用（APK）

安卓工程位于 `android/` 子目录，由 Capacitor 管理。流程：先构建网页 → 同步进安卓 → 用 Gradle / Android Studio 出包。

### 方式 A：Android Studio（推荐，最稳）

```bash
# 1. 构建网页并同步进安卓工程
npm run build:android        # = vite build && npx cap sync android

# 2. 打开 Android Studio
npx cap open android
```

在 Android Studio 中：
- **调试包**：菜单 `Build → Build Bundle(s) / APK(s) → Build APK(s)`
  → 产物 `android/app/build/outputs/apk/debug/app-debug.apk`
- **正式发布包**：`Build → Generate Signed Bundle / APK`，需提前准备 keystore 并在 `android/app/build.gradle` 配置 `signingConfigs`。

### 方式 B：纯命令行（无需 Android Studio）

```bash
npm run build:android
```

随后调用本地缓存的 Gradle 二进制（已绕过联网下载）执行打包：
```bash
set JAVA_HOME=C:\Program Files\Java\jdk-21
"C:/Users/28166/.gradle/wrapper/dists/gradle-8.11.1-all/<hash>/gradle-8.11.1/bin/gradle" -p android assembleDebug
```
> `<hash>` 为随机目录名，用资源管理器打开上述路径即可看到。
> 也可双击项目根目录的 `app一键打包.bat` 一键完成全流程。

**安装到手机（可选）**
```bash
adb install -r android/app/build/outputs/apk/debug/app-debug.apk
```
（手机需开启 USB 调试）

## 六、常见问题

1. **安卓构建报 `invalid source release 21`** → JDK 版本不对，确认 `JAVA_HOME` 指向 JDK 21。
2. **`./gradlew` 卡在下载 Gradle** → 本机无法访问 `services.gradle.org`；改用已缓存的 Gradle 二进制（见方式 B）或配置国内镜像源。
3. **`local.properties` 路径报错** → `sdk.dir` 必须使用正斜杠，反斜杠会被视为非法转义。
4. **离线打开图片不显示** → 确认 `public/sprites/` 存在 1025 张 PNG，且 `spriteFor()` 使用相对路径。
5. **网页部署后白屏** → 检查静态托管是否以子路径 served；本项目 `base: './'` 已兼容大多数场景。

## 八、数据 Excel 往返编辑（批量改数据）

数据表（宝可梦 / 道具 / 招式 / 特性 / 天气 / 场地 / 异常状态 / 小工具 / 属性相克 / 防御矩阵）支持导出成 Excel 编辑、再导回 App 使用的 TS 格式，方便非开发者批量修改中文名、描述、属性等。共 10 个 Sheet，全部使用中文命名。

### 依赖
首次使用需安装 `exceljs`（已在项目中）：
`npm install -D exceljs`

### 命令
```bash
# 1) 导出：把现有 TS 数据写成一个多 Sheet 的 xlsx
node scripts/data-excel.mjs export
#   → 生成 data-excel/宝可梦小图鉴_数据.xlsx

# 2) 在 Excel 里编辑（每表一个 Sheet，可编辑列即建议修改项）
#    注意「英文名」为主键，勿重复或留空；数组字段（属性/相关特性等）用逗号分隔

# 3) 导回：把 xlsx 写回对应 TS 文件（覆盖前自动生成 .bak 备份）
node scripts/data-excel.mjs import
```

### 一键操作（双击 bat，免敲命令）
`data-excel/` 目录内置两个 bat，双击即用：
- **`数据转xlsx.bat`** —— 执行上面的「导出」步骤。
- **`数据转回ts.bat`** —— 执行上面的「导回」步骤（覆盖前自动 .bak 备份）。

> ⚠️ 导出前请先**关闭 Excel 中已打开的 `宝可梦小图鉴_数据.xlsx`**。该文件被 Excel 占用时 Windows 会加锁，导出脚本会报错并提示「请先关闭 Excel 再运行导出」。导回（写 TS）不受此影响。
```

### 说明
- 导出文件：`data-excel/宝可梦小图鉴_数据.xlsx`，每个数据表一个 Sheet，全部中文命名：
  **宝可梦 / 道具 / 招式 / 特性 / 天气 / 场地 / 异常状态 / 小工具 / 属性相克 / 防御矩阵**。
- 招式表所在 `pokemon-moves.ts` 含 `MOVE_DB` + `POKEMON_MOVES` 两个导出，导入时只更新 `MOVE_DB`，`POKEMON_MOVES` 原样保留，不会误删每只宝可梦的技能引用。
- **宝可梦 Sheet 是「复合表」**（`composite: true`）：一行数据同时来自 `pokemon-db.ts`（基础信息 + 世代）与 `pokemon-moves.ts`（技能）。因此**导回时会同时写回这两个 TS 文件**——`pokemon-db.ts` 重建 `POKEMON_DB`（连同 `POKEMON_BY_ID` / `POKEMON_TOTAL` 派生），`pokemon-moves.ts` 只更新 `POKEMON_MOVES`（每只宝可梦的技能引用），`MOVE_DB` 原样保留。宝可梦 Sheet 现共 16 列，含以下关键列：
  - **世代**（数字，1-9）：按编号区间自动归属（第 1 代 1–151、第 2 代 152–251 …… 第 9 代 906–1025），可在 Excel 中手动调整。该值会在详情页以淡色标签「第 N 代」显示，在招式查询中同样以「第 N 代」标签标注（**仅标签、无特殊配色**）。
  - **特性** / **隐藏特性**：均为逗号分隔的特性字符串，分别对应 `abilities` 与 `hiddenAbilities`；隐藏特性为空表示该宝可梦无隐藏特性。详情页中「普通特性」与「隐藏特性」**样式一致、不做额外配色**，仅以文案小节区分。
  - **升级技能**：升级学会的招式，Excel 单元格格式为 `Lv16 撞击, Lv23 吼叫`（「Lv等级 + 空格 + 中文招式名」，多条用逗号分隔）；导入时解析 `Lv` 数字与招式名，映射回英文 key 写入 `levelMoves`。
  - **机器技能** / **遗传技能**：机器（学习机 / 招式记录）与遗传学会的招式，格式为逗号分隔的中文招式名（如 `撞击, 吼叫`）；导入时按中文名映射回英文 key，分别写入 `machineMoves` / `eggMoves`。
  - 上述三类技能列在 Excel 中宽度较大（50 字符），便于阅读完整招式串。
- 隐藏特性的数据来源：`build-pokemon-db.mjs` 在抓取时已记录 PokeAPI 的 `is_hidden`；若需对已有 `pokemon-db.ts` 补抓（不重抓其它字段），运行 `node scripts/build-hidden-abilities.mjs` 即可，原文件自动备份为 `pokemon-db.ts.bak`。
- 工具表 `tools.ts` 含多个导出（`TOOLS` / `TOOL_MAP` / `TYPE_MATCHUP_DATA` / `DEFENSE_MATRIX` / `CHART_ORDER`）及两个工具函数。导入时按「同一文件整体重建」处理：小工具/属性相克/防御矩阵三个 Sheet 的数据分别写回对应常量，`TOOL_MAP` 由 `TOOLS` 派生，`CHART_ORDER` 原样保留，末尾 `getMatchupCell` / `getMatchupCellClass` 两个函数逐字保留，不会丢失。
- 防御矩阵 Sheet 为 18×18 倍率矩阵（行=攻击属性，列=防御属性；2=绝佳、1=一般、½=不佳、0=无效），可在 Excel 中直接修正数值。
- 表格字段定义（Sheet 名、列映射、可编辑项）集中在 `scripts/excel-config.mjs`，增删表或调整列都在那里改。
- 导入后建议跑一次 `npm run build` 确认类型无误；如需还原，可用各 `.bak` 备份。
