## 修复 Android 侧滑返回失效问题

**现象**：打包 APK 后，从屏幕边缘侧滑无法返回上一页。

**根因**（两点）：
1. `AndroidManifest.xml` 缺少 `android:enableOnBackInvokedCallback="true"` —— Android 13+ (API 33+) 预测式返回手势（侧滑返回动画）必须显式开启，否则系统不将返回手势派发给 WebView。项目 targetSdkVersion=35，必须设置。
2. Web 层没有 `backButton` 事件监听 —— 未安装 `@capacitor/app` 插件，Capacitor 默认行为 `webView.goBack()` 在 hash 路由 + 部分国产 ROM 下不可靠。

**修复内容**：
- `android/app/src/main/AndroidManifest.xml`：`<application>` 标签添加 `android:enableOnBackInvokedCallback="true"`
- 安装 `@capacitor/app@^7.0.0`（注意：不能装 v8，v8 要求 `@capacitor/core@>=8`，与项目 Capacitor 7 冲突）
- `src/main.ts`：注册 `CapApp.addListener('backButton', ...)` —— 有历史则 `window.history.back()`，根页面则 `CapApp.exitApp()`；用 `Capacitor.isNativePlatform()` 守卫，仅原生平台注册
- 已 `npm run build` + `npx cap sync android`，同步成功（Capacitor 识别到 `@capacitor/app@7.1.2`）

**改动的文件**：
- `android/app/src/main/AndroidManifest.xml`
- `src/main.ts`
- `package.json` / `package-lock.json`（新增依赖）
