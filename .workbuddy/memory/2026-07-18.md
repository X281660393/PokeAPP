## 修复：招式/特性/天气/场地/异常状态 点「相关」后返回丢失详情
- 现象：在 `/tools/ability-search`、`/tools/move-search`、CodexSearch（天气/场地/异常状态）中，点列表项进入详情后，再点「相关」链接跳到另一项，按返回键（ToolDetail 的 `router.back()`）会直接回到列表页，跳过中间详情。
- 根因：这些页的「选中态」都只存在组件内 `selectedKey` 局部状态，**未写入 URL**。点相关链接 `router.push` 到另一路由后，再 `router.back()` 回到源路由时组件重新挂载、`selectedKey` 归零 → 只剩列表。
- 路由注意：`/tools/ability-search`、`/tools/move-search` 并非独立路由，实际匹配 `/tools/:toolId`（由 `ToolDetail.vue` 按 `toolId` 切换渲染对应子组件）；CodexSearch 经 WeatherSearch/TerrainSearch/StatusSearch 复用。
- 修复（三个文件）：
  - `AbilitySearch.vue` / `MoveSearch.vue`：`openAbility`/`openMove` 改为 `selectedKey.value = key; router.push({ query: { key } })`；原 `?key=` 深链 `watch` 补上「key 缺失即 `selectedKey=null`」分支，使返回列表态也能正确还原。
  - `CodexSearch.vue`：新增 `useRoute` 与 `selectItem(key)`（`selectedKey.value = key; router.push({ query: { sel: key } })`，用 `sel` 区分自身选中与对外 `?key=` 深链）；`watch(route.query.sel)` 同步选中态；模板列表项 `@click` 由 `selectedKey = i.key` 改为 `selectItem(i.key)`。
- 行为：点列表项用 `push` 写入 URL（生成历史记录），从相关链接返回时即可逐步还原每个详情；返回列表时 `?key/?sel` 缺失会把选中清空。
- 验证：`vue-tsc -b` 类型检查通过，无报错；dev server 8081 返回 200（HMR 生效）。
