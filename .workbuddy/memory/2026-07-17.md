### 再次修正：app一键打包.bat 版本号读取仍失败（"文件名语法不正确"）
- 现象：用户运行后报 `The filename, directory name, or volume label syntax is incorrect.`，随后 `[ERR] 无法从 src/config/index.ts 读取版本号`。
- 根因：`for /f` 的 inline command 中包含多组双引号路径，cmd 在解析外层单引号与内层双引号时产生语法错误，导致 Node 命令根本没执行；`if not defined VER` 被触发报错。
- 修正：去掉 `for /f`，改为 Node 直接把版本号写到临时文件，bat 用 `set /p` 读取：
  ```bat
  set "VER_FILE=%TEMP%\poke-version-%RANDOM%.tmp"
  "%NODE_DIR%\node.exe" "%~dp0scripts\get-version.mjs" "%~dp0src\config\index.ts" > "%VER_FILE%" 2>&1
  if %errorlevel% neq 0 ( ... 报错并显示 stderr ... )
  set /p VER=<"%VER_FILE%"
  del /f /q "%VER_FILE%"
  ```
  这样完全绕开 cmd 对引号的解析问题，且 `stderr` 也捕获到文件便于排错。
- 验证：本地 Node 运行 `scripts/get-version.mjs` 仍输出 `1.0.01`；bat 中的新命令结构在 cmd 下不再依赖 `for /f` 的复杂引号解析。

## 路由滚动：图鉴进入详情不回顶
- 需求：app 里从图鉴列表点进宝可梦详情，详情页应从顶部开始显示，而不是沿用列表的滚动位置。
- 根因：`src/router/index.ts` 的 `createRouter` 没有配置 `scrollBehavior`，而项目用 hash 路由 + keep-alive + window 滚动（`.page-container` 不创建自身滚动容器，滚动交给 document）。因此前进导航时窗口滚动位置被带到详情页，详情从中间显示。
- 修正：给 router 增加 `scrollBehavior(to, from, savedPosition)`：
  - `savedPosition` 存在（popstate/返回）→ 返回 `savedPosition`（恢复离开前位置）；
  - 否则（前进 push，如图鉴→详情、点招式/特性跳小工具）→ 返回 `{ top: 0 }` 回顶。
  - vue-router 在设置 scrollBehavior 时会自动 `history.scrollRestoration = 'manual'`。
- 互补关系：图鉴 `Pokedex.vue` 本身有 `onActivated/onDeactivated` 的滚动缓存（保存/恢复 window.scrollY），负责「返回列表恢复位置」与 tab 切换；与全局 scrollBehavior 不冲突（返回时两者都滚到同一位置）。
- 验证：`npx vue-tsc -b` EXIT=0。行为：进入任意详情/子页均回顶；从详情返回图鉴仍恢复原位。

## 修复 data-excel/Excel 0KB 残留（真因：bat title 行的 > 被 cmd 当重定向）
- 现象：`data-excel/` 每次双击 `转Excel.bat` 后都会多出一个 0 字节、无扩展名的文件 `Excel`，时间戳与 `宝可梦小图鉴_数据.xlsx` 完全一致。
- 第一次误判：以为是历史残留，删了 + 加 .gitignore 即可。结果用户再跑 bat 又出现，证明是脚本/bat 每次生成的。
- 真因：`转Excel.bat` 第 2 行 `title 数据(TS) -> Excel 导出` 与 `转回Ts.bat` 第 2 行 `title Excel -> 数据(TS) 导回` 里的 `>` 被 cmd 当成**输出重定向符**，把 title 命令（无 stdout）重定向到下一个 token 命名的文件 → 创建 0 字节空文件 `Excel`（/`数据(TS)`）。这是 cmd `title` 命令的经典坑。
- 排除：通读 `scripts/data-excel.mjs` 全文 + `node_modules/exceljs/dist/exceljs.js` 的 ZipWriter 实现，确认 ExcelJS 用 JSZip 在内存压缩、通过 `fs.createWriteStream` 直接写出，**不会创建临时文件**；`mkdirSync` 也只建 `data-excel/` 目录。所以 0 字节 `Excel` 文件 100% 来自 bat 的 title 行。
- 修复：用 Python 以 GBK 精确替换两个 bat 的 title 行，把 `>` 转义为 `^>`：
  - `title 数据(TS) -> Excel 导出` → `title 数据(TS) -^> Excel 导出`
  - `title Excel -> 数据(TS) 导回` → `title Excel -^> 数据(TS) 导回`
  （cmd 会把 `^>` 还原成 `>` 显示在窗口标题里，视觉无变化；但不再触发重定向）
- 验证：删 `Excel` 后在 cmd 里跑修复后的 title 行，`data-excel/` 不再生成 `Excel` 文件。
- 教训：**Windows bat 的 `title` 命令后若含 `>` `<` `|` `&` 等特殊字符必须转义**（用 `^>` `^<` `^|` `^&`），否则会被 cmd 解释为重定向/管道/命令连接，副作用是创建空文件。
- 顺手：`.gitignore` 已加 `/data-excel/Excel`（这条规则其实可以保留作为兜底，但根因已除）。

### 修正：数据转回ts.bat 双击失败（路径被 Python 转义破坏）
- 现象：用户双击 `数据转回ts.bat` 后 cmd 报「系统找不到指定的路径」「not/晚?cd/ns 不是内部或外部命令」等乱码，随后窗口进入 Vite dev（用户可能在失败后手动跑了 dev）。
- 根因：之前用 Python 非 raw string 写 bat 时，路径中的 `\b` 被 Python 当成退格（`\b` → backspace 0x08）、`\n` 被当成换行、`\v` 被当成垂直制表符，导致 `NODE_EXE` 路径变成乱码、行解析错位，cmd 把后面的文本误当命令执行。
- 修正：改用 Python **raw string** (`r'''...'''`) 写 bat，保留单反斜杠；再统一把换行转成 CRLF（`replace('\n','\r\n')`）。
- 验证：`xxd` 检查 `NODE_EXE` 行字节现为 `%USERPROFILE%\.workbuddy\binaries\node\versions\22.22.2\node.exe`（正常反斜杠）；两个 bat 均为 GBK + CRLF。直接运行 `node scripts/data-excel.mjs import` 成功读取 10 个 Sheet 并写回；`vue-tsc` EXIT=0。`.bak` 已清理。
- 注意：若用户已在 Excel 里编辑过 xlsx，**关闭 Excel 后双击 `数据转回ts.bat`** 即可正常导回；如仍报错可先用命令 `node scripts/data-excel.mjs import` 作为兜底。
